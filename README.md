# TimeTrack Analytics

Проект для анализа распределения времени на основе личных логов активности.



## Архитектура проекта и сбор данных

Проект представляет собой автоматизированную систему сквозного трекинга времени и анализа личной продуктивности. Сбор и визуализация данных разделены на три ключевых этапа:

1. **Сбор данных (Android / MacroDroid):**
   * На смартфоне настроен кастомный макрос в приложении **MacroDroid**.
   * Каждые 15 минут с 8:15 до 00:05 на экране телефона всплывает диалоговое окно с предложенными вариантами активностей за прошедший интервал.
   * Выбранный вариант вместе с меткой времени автоматически отправляется по API в облачную таблицу.

2. **Хранилище данных (Google Sheets API):**
   * Центральным хранилищем выступает **Google Sheets**.
   * Бэкенд-часть приложения взаимодействует с таблицей в режиме реального времени через библиотеку `gspread` и сервисный аккаунт Google Cloud.

3. **Бэкенд и Аналитическая панель (FastAPI + Streamlit):**
   * **FastAPI (Backend):** Выступает в роли промежуточного API - забирает сырые логи из Google Sheets.
   * **Streamlit (Frontend):** Интерактивный дашборд, который обращается к бэкенду, рассчитывает KPI продуктивности и строит графики распределения времени по категориям.

---

## Технологический стек

* **Data Capture:** Android + MacroDroid Macros
* **Storage:** Google Sheets API (`gspread`)
* **Backend:** Python 3.11, FastAPI, Uvicorn
* **Frontend / BI:** Streamlit, Altair / Plotly
* **DevOps:** Docker, Docker Compose




## Запуск через Docker
1. Выполните команду в корне проекта: `docker-compose up -d --build`
2. Откройте в браузере: http://localhost:8501
